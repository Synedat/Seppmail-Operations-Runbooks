# Workshop kit

## Workshop outcome
Help participants understand **Operational runbooks for incidents, changes and evidence-oriented execution** and identify a path from initial exploration to reusable operations.

## Target audience
Operations leads, support teams and managed service teams.

## Suggested agenda
1. Context and problem statement
2. Architecture and trust boundaries
3. Permissions and least-privilege considerations
4. Example run or demo output
5. Operations, evidence and rollback approach
6. Extension backlog and next steps

## Materials
- README and quick start
- `docs/ARCHITECTURE.md`
- `docs/RBAC-AND-PERMISSIONS.md`
- `docs/SECURITY-AND-COMPLIANCE.md`
- `docs/EXAMPLE-OUTPUTS.md`
- `examples/demo-walkthrough.md`

## Good workshop questions
- Which parts should remain read-only first?
- Which data must be masked in public demos?
- Which control owners need evidence from this workflow?
- Which operational metrics should be tracked after go-live?

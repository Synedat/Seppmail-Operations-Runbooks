# Implementation playbook v10

## Start pattern
1. Read the README and architecture notes.
2. Review permissions and security/compliance mapping.
3. Run one low-risk example or inspect the demo content.
4. Decide which parts remain public baseline and which become project-specific overlays.
5. Capture evidence, export outputs and document extension points.

## Delivery-friendly sequence
- discovery
- controlled validation
- architecture review
- pilot implementation
- operationalization
- evidence and handover

## Reuse guidance
Treat this repository as a well-documented starter, not a drop-in production promise. Tailor naming, secrets, topology, monitoring and approval points to the target environment.

# Demo scenarios

## Scenario 1: Quick orientation

- Open the README and identify the supported use cases
- Review the architecture diagram and trust boundaries
- Run one example script in a lab or dry-run context
- Review the generated evidence or outputs

## Scenario 2: Operational walkthrough

- Validate prerequisites and permissions
- Execute a representative example or runbook step
- Capture logs, outputs and screenshots
- Compare outcomes with the expected control mapping

## Scenario 3: Extension workshop

- Review `docs/EXTENSIONS-AND-ROADMAP.md`
- Select one missing automation or reporting gap
- Prototype it under `examples/` or a new module folder
- Document assumptions, rollback and observability impact

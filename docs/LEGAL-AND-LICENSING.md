# Legal and Licensing Notes

## Purpose

This repository provides public implementation guidance for **Seppmail-Operations-Runbooks**. It is structured to be useful for practitioners while remaining clear about authorship, licensing and upstream references.

## Practical guidance

- Keep third-party code and copied vendor material clearly separated from original Synedat-authored content.
- Prefer linking to upstream SEPPmail documentation instead of copying long extracts.
- Preserve attribution when adapting snippets from upstream vendor examples.
- Record provenance for diagrams, templates and screenshots before publishing.

## Recommended repository hygiene

- Keep `LICENSE`, `NOTICE`, `SECURITY.md`, `SUPPORT.md` and `CODE_OF_CONDUCT.md` in the repository root.
- Document external dependencies and modules in the README and release notes.
- Track whether examples are demonstrational, production-ready or starter content.

# GitHub Discussions Setup

## Why enable discussions

Discussions help convert one-off support traffic into discoverable community knowledge for **Seppmail-Operations-Runbooks**.

## Suggested categories

- Announcements
- Q&A
- Ideas
- Show and tell
- Operations and troubleshooting
- Architecture and security

## Moderation approach

- Redirect vulnerability reports away from public threads.
- Move generic support requests into reusable answers and FAQs.
- Pin a starter thread with supported scenarios, disclaimers and links to official SEPPmail documentation.

## Starter prompts

- How are you using this repository in Exchange Online or Azure?
- Which runbooks or examples would improve your rollout?
- Which compliance mapping or evidence pattern would be most useful next?

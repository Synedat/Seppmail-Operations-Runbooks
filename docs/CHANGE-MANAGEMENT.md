# Change management

## Recommended minimum process

1. capture current state and context
2. describe the intended change and the rollback path
3. peer-review permissions and blast radius
4. execute within an approved window
5. validate outcome and capture evidence
6. update the operating documentation

## Pre-change checklist

- scope and owner confirmed
- rollback documented
- evidence location defined
- approver identified
- smoke-test steps prepared

## Post-change checklist

- outcome validated
- unexpected side effects checked
- evidence attached to the ticket
- remaining follow-up items logged

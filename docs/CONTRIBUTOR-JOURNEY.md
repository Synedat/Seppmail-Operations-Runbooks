# Contributor journey

## Good first contribution ideas

- Improve a diagram or example comment
- Add validation or error handling to an existing script
- Extend troubleshooting notes with real-world findings
- Add a missing permission, dependency or rollback note

## Suggested contribution flow

1. Open an issue
2. Link the intended change to a use case or operational pain point
3. Update docs and examples together
4. Include evidence, screenshots or before/after outputs where useful

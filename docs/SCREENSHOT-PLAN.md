# Screenshot plan

Use this checklist to create more realistic visuals once the repository is published.

## Recommended screenshots
- repository landing page with badges and hero image
- code example view or demo data file
- architecture diagram or deployment plan
- changelog / release page
- wiki or GitHub Pages landing page

## Redaction guidance
- remove real tenant names and IPs
- mask tokens, secrets and certificates
- avoid screenshots that reveal internal hostnames or customer identifiers
